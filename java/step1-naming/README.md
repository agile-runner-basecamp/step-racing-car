# Step 1: 이름 짓기와 매직 넘버 (Naming & Magic Number)

이 코드는 **기능적으로 완벽하게 동작하며 모든 테스트를 통과**합니다.
하지만 코드를 처음 보는 동료 개발자는 이 코드가 도대체 무슨 일을 하는지 이해하기 매우 어렵습니다.

## 🎯 학습 목표
- 변수명과 메서드명에 의도를 담는 연습을 합니다.
- 코드 중간에 흩뿌려진 '의미를 알 수 없는 숫자(매직 넘버)'를 상수로 추출합니다.

## 📝 요구 사항
1. `BadRacer.java` / `badRacer.ts` 코드를 열고, 메서드명(`doM`, `doW`, `chk`)과 변수명(`r`, `m`, `c`, `w` 등)을 의미 있는 이름으로 변경하세요.
2. 코드 안에 하드코딩된 숫자들(예: `4`, `5`, `0`, `9`)의 의미를 파악하고 상수로 추출하세요.
3. **테스트 코드도 개선하세요.** "테스트는 문서다"라는 관점에서:
   - 메서드명: `t1` → `moveForward_whenRandomValueIsAboveThreshold` (영어)
   - `@DisplayName` / `it`: `"테스트1"` → `"랜덤 값이 4 이상이면 전진한다"` (한글 설명)
4. 리팩토링 중간중간 테스트를 실행하여 **Green이 유지되는지** 확인하세요!

## ✅ 달성 기준
- [ ] 모든 메서드명이 "이 메서드가 무엇을 하는지" 설명하고 있는가?
- [ ] 모든 변수명이 "이 변수가 무엇을 담고 있는지" 설명하고 있는가?
- [ ] 매직 넘버가 상수로 추출되었는가?
- [ ] 테스트의 `@DisplayName` / `it`이 한글 문장으로 동작을 설명하는가?
- [ ] 모든 테스트가 Green인가?
