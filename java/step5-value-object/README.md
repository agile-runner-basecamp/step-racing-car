# Step 5: 원시값 포장 (Value Object)

이 코드는 **기능적으로 완벽하게 동작하며 테스트를 통과**합니다.
하지만 자동차의 이름(`String`)과 위치(`int`)가 **날것 그대로** 사용되고 있어, 검증 로직이 생성자에 몰려 있고 도메인의 의미가 드러나지 않습니다.

## 🎯 학습 목표
- 원시 타입을 의미 있는 객체로 감싸는 **Value Object** 패턴을 체득합니다.
- "이름은 5자 이하", "위치는 0 이상" 같은 규칙을 **객체 스스로가 보장**하도록 만듭니다.

## 📝 요구 사항
1. `RawCar.java` / `rawCar.ts`의 `String name`을 `CarName` 객체로 포장하세요.
2. `int position`을 `Position` 객체로 포장하세요.
3. Value Object의 3대 특성을 지키세요:
   - **불변성**: `final` 필드 / `readonly`, setter 없음
   - **동등성**: `equals()`와 `hashCode()` 재정의 (Java) / 값 비교 메서드 (TS)
   - **유효성 검증**: 생성자에서 값의 유효성 보장
4. 포장 후 새로운 테스트를 추가하세요:
   - CarName의 동등성 테스트
   - Position이 음수가 될 수 없다는 테스트

## ✅ 달성 기준
- [ ] `CarName` 클래스가 존재하고, 5자 초과 시 생성 단계에서 예외가 발생하는가?
- [ ] `Position` 클래스가 존재하고, 0 이상의 값만 허용하는가?
- [ ] Value Object가 불변(immutable)인가?
- [ ] `Car` 클래스에서 `String`과 `int`가 직접 사용되지 않는가?
- [ ] 모든 테스트가 Green인가?
