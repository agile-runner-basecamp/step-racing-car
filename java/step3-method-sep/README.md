# Step 3: 메서드 분리 (한 메서드 = 한 가지 일)

이 코드는 **기능적으로 완벽하게 동작하며 테스트를 통과**합니다.
하지만 입력 파싱, 검증, 라운드 실행, 우승자 판정, 결과 포맷팅까지 **60줄짜리 하나의 거대한 메서드** 안에 엉켜 있습니다.

## 🎯 학습 목표
- "한 메서드는 한 가지 일만 해야 한다"는 원칙을 체득합니다.
- 리팩토링 후 메서드를 읽었을 때 **"목차를 읽는 것처럼"** 흐름이 한눈에 파악되어야 합니다.

## 📝 요구 사항
1. `MassiveRace.java` / `massiveRace.ts`의 `race()` 메서드를 여러 개의 의미 있는 메서드로 분리하세요.
2. 예: `validateInput()`, `parseNames()`, `playRounds()`, `findWinners()`, `formatResult()` 등
3. 분리 후 `race()` 메서드를 읽으면 추상화 레벨이 동등하게 유지되어야 합니다.
4. 💡 **보너스:** 매직 넘버 상수 추출도 함께 적용해 보세요.
5. 테스트를 실행하여 동작이 깨지지 않았는지 확인하세요!

## ✅ 달성 기준
- [ ] `race()` 메서드 본문이 5~10줄 이내로, 각 줄이 무엇을 하는지 한눈에 파악되는가?
- [ ] 분리된 각 메서드가 하나의 역할만 수행하는가?
- [ ] 추상화 레벨이 동등한가? (높은 수준 사이에 낮은 수준이 섞여있지 않은가?)
- [ ] 모든 테스트가 Green인가?
