# Step 4: 관심사 분리 (도메인 ↔ UI 분리)

이 코드는 **기능적으로 완벽하게 동작**하며, early return과 메서드 분리도 적용되어 있습니다.
하지만 치명적인 문제가 있습니다! **경주 로직 한가운데에 `System.out.println` / `console.log`가 강하게 결합되어 있습니다.**

이로 인해:
1. 경주 결과를 자동화된 단위 테스트로 검증하기가 극도로 까다롭습니다.
2. 나중에 이 게임을 웹이나 앱으로 이식하려면 코드를 전부 다시 짜야 합니다.

## 🎯 학습 목표
- Model(도메인)과 View(UI)를 분리하는 감각을 익힙니다.
- 단위 테스트하기 쉬운 "순수한 로직"을 만듭니다.

## 📝 요구 사항
1. `ConsoleRace.java` / `consoleRace.ts`에서 `System.out` / `console.log`를 걷어내세요.
2. 경주 로직은 반환값을 가진 순수한 메서드로 개선하세요.
3. 출력 기능은 별도의 View 클래스나 함수로 분리하세요.
4. 순수해진 도메인 로직에 대해 단언(assertThat/expect)이 가능한 테스트를 작성하세요.

## ✅ 달성 기준
- [ ] 도메인 로직 클래스에 `System.out` / `console.log`가 하나도 없는가?
- [ ] 경주 결과를 반환값으로 검증할 수 있는 테스트가 작성되었는가?
- [ ] 우승자를 반환값으로 검증할 수 있는 테스트가 작성되었는가?
- [ ] 모든 테스트가 Green인가?
